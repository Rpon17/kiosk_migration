# 🛒 Kiosk Migration Project (POS & Kiosk Management System)

파이썬 GUI 환경(`PySide6`)으로 구축된 키오스크/POS 관리 시스템을 모던 웹 프론트엔드(`React`) 환경으로 마이그레이션하는 프로젝트입니다.

---

## 🛠️ 기술 스택 (Tech Stack)

### Backend & Database
- **Framework:** FastAPI (Python)
- **Database:** Supabase (PostgreSQL)
- **ORM:** SQLAlchemy

### Frontend
- **As-Is (기존):** PySide6 (Desktop GUI)
- **To-Be (목표):** React (Typescript 기반 web)

---

## 🗂️ 데이터베이스 구조 및 핵심 기능

시스템은 유기적으로 연결된 **4개의 핵심 엔티티(테이블)**를 기준으로 관리됩니다.

### 1. 매장 관리 (`Store`)
- 매장 생성 및 전체 매장 리스트 조회

### 2. 카테고리 관리 (`Category`)
- 매장별 하위 카테고리 생성 및 조회

### 3. 상품 관리 (`Product`)
- 카테고리별 상품 등록 및 상세 조회

### 4. 재고 관리 (`Stock`)
- 상품별 실시간 재고 생성, 수량 추가 및 변경(갱신)

---

## 💡 한눈에 보는 데이터 흐름 예시

본 시스템은 매장부터 재고까지 물 흐르듯 이어지는 **계층형 구조**를 가집니다.

> 🏢 **1. 매장 관리:** `매드구스 전농점`을 등록하고 선택합니다.
> 📁 **2. 카테고리 관리:** 해당 매장 하위에 `지우개`와 `스티커` 카테고리를 분류합니다.
> 🎁 **3. 상품 관리:** 스티커 카테고리 내에 `거위 스티커`와 `고양이 스티커` 상품을 등록합니다.
> 📦 **4. 재고 관리:** `거위 스티커` 상품의 현재 실재고를 `30개`로 생성하고 변경 관리합니다.

---

## 🎯 마이그레이션 목표 (Migration Roadmap)

1. **데스크톱에서 웹으로의 전환:** 장소와 기기에 구애받지 않는 크로스 플랫폼 관리를 위해 데스크톱 App을 웹 App(React) 환경으로 전환합니다.
2. **상태 관리 및 UI 컴포넌트화:** PySide6의 `Signal/Slot` 기반 이벤트 아키텍처를 React의 상태 관리(State) 및 Hook 메커니즘으로 이식합니다.
3. **API 레이어 유지:** 동일한 FastAPI 백엔드 명세를 활용하여 프론트엔드 엔진만 안정적으로 교체합니다.
